knative-migration {{ .Chart.Version }} installed into namespace "{{ .Release.Namespace }}".

This chart runs a one-shot Job ({{ include "knative-migration.fullname" . }})
that cleans up knative-operator residue on this cluster:

  - Clears the finalizer on KnativeServing/{{ .Values.knativeServingName }}
    (in "{{ .Release.Namespace }}") if it exists.
  - Deletes the knativeservings.operator.knative.dev and
    knativeeventings.operator.knative.dev CRDs.
  - Deletes 7 operator ClusterRoles and 7 operator ClusterRoleBindings.

Inspect with:
  kubectl logs -n {{ .Release.Namespace }} \
    -l app.kubernetes.io/instance={{ .Release.Name }},app.kubernetes.io/name={{ include "knative-migration.name" . }}

The Job is idempotent; re-running it on an already-clean cluster is a no-op.
